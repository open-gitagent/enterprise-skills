## Role
Continuously identify, assess, implement, and report regulatory changes through an obligations register, horizon scanning, gap analysis, and layered compliance reporting.

## Regulatory Change Triage Matrix

| Tier | Profile | Action |
|---|---|---|
| Tier 1 (Critical) | New licensing, material capital/operational changes, penalties exceeding risk appetite | Board/C-suite notification within 48 hours |
| Tier 2 (Significant) | Policy/procedure amendments, system modifications, staff retraining | Implementation within defined window |
| Tier 3 (Moderate) | Documentation updates, minor process adjustments, enhanced monitoring parameters | Update documentation |
| Tier 4 (Low) | No current action required | Informational tracking; monitor for escalation |

## Program Maturity Model

| Level | Description | Key Indicators |
|---|---|---|
| 1 — Ad Hoc | Reactive, individual-dependent | No obligations register; manual tracking; gaps discovered in examinations |
| 2 — Developing | Basic processes exist | Partial register; irregular scanning; gap analysis only pre-audit |
| 3 — Defined | Standardized workflow | Complete register; systematic scanning; documented gap analysis; regular reporting |
| 4 — Managed | Metrics-driven | KPIs tracked; timelines measured; testing integrated; board reporting established |
| 5 — Optimized | Predictive/adaptive | Automated RegTech; predictive analytics; continuous monitoring; real-time dashboards; proactive regulator engagement |

## Key Performance Indicators

| Metric | Target |
|---|---|
| Gap assessment completion rate (Tier 1/2) | ≥95% within SLA |
| Implementation completion by regulatory deadline | 100% mandatory deadlines |
| Compliance testing deficiency rate | <5% of controls tested |

| Proactive compliance ratio | >60% |

## Process

1. **Establish Obligations Register** — Build structured inventory: all applicable laws/regs/rules/voluntary commitments mapped to business units, products, jurisdictions, implementing controls, named owners (not departments), effective dates, phase-in schedules; per ISO 37301:2021 §4.5
2. **Regulatory Horizon Scanning** — Monitor three horizons: Immediate (0-90 days): final rules, enforcement actions, no-action letters, exam priorities; Near-term (90 days–12 months): proposed rules, comment periods, consultation papers; Strategic (12+ months): legislative proposals, international convergence; sources: Federal Register, FINRA Regulatory Notices, SEC Bulletins, OCC Bulletins, CFPB Supervisory Highlights, FinCEN Advisories; RegTech platforms filter by jurisdiction/taxonomy (250+ daily global changes per Thomson Reuters)
3. **Perform Impact Assessment** — Map new/amended requirements against obligations register; identify control gaps; quantify implementation effort (hours, system changes, budget); assess enforcement risk; document methodology, participants, conclusions
4. **Execute Gap Analysis** — Control mapping: Compliant/Partially Compliant/Non-Compliant per requirement element; root cause: policy absence/procedural weakness/system limitation/training gap/resource constraint; materiality score: likelihood × consequence; flag cross-functional dependencies
5. **Develop Implementation Plan** — Prioritize by regulatory deadline, enforcement risk, dependency chain; assign named owners with accountability acceptance; define measurable milestones with specific completion criteria and deadlines; establish escalation triggers; budget technology/counsel/training
6. **Implement & Test** — First-line business units modify processes/systems/training; second-line compliance conducts independent testing (transaction testing, sample review, process walkthroughs, automated monitoring); document all evidence for exam readiness
7. **Compliance Reporting** — Board/Audit Committee: quarterly dashboard (change volume, open gaps by severity, implementation status, emerging risks, resource adequacy); Senior Management: monthly operational; Regulators: exam-ready packages; FINRA Rule 3120 annual; SEC Form ADV ≤90 days post fiscal year-end; SOX 404 with 10-K; OFAC annually; SAR ≤30 days; complete audit trail for each regulatory change
8. **Continuous Improvement** — Post-implementation reviews for significant changes; track mean time detection→implementation, gap closure rates, testing deficiency rates, exam findings; benchmark vs. peers; update framework based on lessons learned

## Glossary

| Term | Definition |
|---|---|
| Obligations Register | Continuously maintained inventory of all applicable laws/regs/rules/commitments mapped to owners, controls, business units |
| Regulatory Horizon Scanning | Systematic forward-looking monitoring of regulators/legislative bodies to detect changes before binding |
| Gap Analysis | Structured comparison of current vs. required: Compliant/Partially Compliant/Non-Compliant per element with root cause |
| Regulatory Change Management (RCM) | End-to-end workflow: Detect → Assess → Assign → Plan → Implement → Test → Report → Archive |
| Three Lines of Defense (3LoD) | 1st line: business execution; 2nd line: compliance/risk oversight; 3rd line: internal audit assurance; IIA 2020 update |
| Control Mapping | Linking regulatory requirements to internal controls (policies, procedures, system configs) enabling gap identification |
| Materiality Threshold | Boundary above which gap/change/exposure requires escalated attention or accelerated remediation |
| RegTech | AI-powered monitoring, automated obligation parsing, workflow management, and control testing technology |
| Compliance Testing | Independent verification controls operate effectively; distinct from monitoring; includes transaction testing, walkthroughs |
| Risk Appetite Statement | Board-approved articulation of acceptable risk types/levels; framework for gap prioritization |
| Enforcement Action Trend Analysis | Monitoring enforcement outcomes to predict examination focus and calibrate internal priorities |
| Self-Reporting Obligation | FINRA Rule 4530, OFAC voluntary self-disclosure; compliance programs must include escalation protocols |
| Supervisory Expectations | Non-binding but enforceable regulator guidance via exam manuals, staff bulletins, speeches |
| IVT (Implementation Verification Testing) | Targeted testing confirming new regulatory change operationalized correctly; conducted post-implementation |
| Compliance Risk Assessment (CRA) | Periodic enterprise-wide evaluation of inherent compliance risk across obligations/products/geographies |
| Remediation Roadmap | Prioritized, time-bound, owner-assigned action plan with milestones, completion criteria, dependencies |
| Regulatory Inventory (LRR) | Law, Rule, Regulation inventory; source of truth for obligations register; maintained in GRC platforms |
| Cooperation Credit | Penalty reduction for voluntary disclosure, investigation cooperation, proactive remediation; SEC FY2024 confirmed benefit |
| Examination Readiness | Complete audit trails, current policies, testing evidence, accessible compliance reporting for regulatory exam |
| Sunset Provisions | Regulatory requirements with expiration/reauthorization dates requiring tracking |
| Cross-Jurisdictional Conflict | Contradicting obligations from different regulators/jurisdictions requiring documented resolution frameworks |
| Proactive Compliance Ratio | % activity on forward-looking vs. reactive remediation; best-in-class target >60% |

## Checklist

- [ ] Obligations register comprehensive, all applicable jurisdictions, validated within past 12 months
- [ ] Horizon scanning covers all three time horizons (immediate/near-term/strategic) with documented source coverage
- [ ] Each regulatory change has documented applicability determination with rationale
- [ ] Impact assessments for Tier 1/2 include specific gap ID, root cause, remediation cost estimates
- [ ] Every identified gap has named individual owner (not department) with documented accountability acceptance
- [ ] Implementation plans contain measurable completion criteria with specific deadlines (not just task descriptions)
- [ ] Second-line independent testing conducted for all implemented regulatory changes; deficiencies tracked to closure
- [ ] Board/audit committee receives quarterly reports disclosing aggregate metrics AND specific material gaps
- [ ] Program assessed against DOJ ECCP criteria (design, implementation, effectiveness) within past 12 months
- [ ] Self-reporting protocols exist and communicated to compliance staff with escalation timelines and counsel engagement
- [ ] Third-party/vendor compliance obligations included in monitoring scope with contractual requirements and periodic verification
- [ ] AI/emerging technology inventoried and assessed for compliance risk per DOJ ECCP Sept 2024
- [ ] Regulatory change detection-to-assessment latency measured, reported, within SLA thresholds
- [ ] Compliance monitoring technology, staffing, and budget adequacy assessed and documented relative to risk profile
- [ ] Post-implementation reviews conducted for material regulatory changes with lessons learned incorporated
